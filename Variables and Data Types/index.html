Variables & Data Types

Common Data Types

  • Number: Used for numeric values and can store both integers and floating point values. You can declare a number variable using the var, let, or const keywords, followed by the variable name and an optional assignment. Supports arithmetic operations (sum, difference, product, quotient) and mathematical functions (Math.sqrt(), Math.pow()).
    var age = 25;
    let temperature = 98.6;
    const PI = 3.14;
  • String: Used for textual data. Strings are declared by enclosing the text in single quotes ('') or double quotes ("").
    var name = 'John';
    let message = "Hello, world!";
    const companyName = 'ABC Corp';
  • Boolean: Used for representing true or false values. Boolean variables can be declared and assigned using the same var, let, or const keywords.
    var isLogged = true;
    let isActive = false;
    const hasPermission = true;
  • Array: Used for storing multiple values in a single variable. Arrays can be declared using square brackets ([]) and can contain elements of any type.
    var numbers = [1, 2, 3, 4, 5];
    let fruits = ['apple', 'banana', 'orange'];
    const colors = ['red', 'green', 'blue'];
  • Object: Used for storing key-value pairs. Objects are declared using curly braces ({}), and properties are defined using a key-value syntax.
    var person = { name: 'John', age: 25 };
    let car = { brand: 'Toyota', model: 'Camry' };
    const employee = { name: 'Jane', id: 12345 };
  • Null: Represents the intentional absence of any object value. It is assigned using the keyword null.
    var data = null;
  • Undefined: Represents an uninitialized or unassigned variable. By default, variables are assigned the value undefined if they are not explicitly given a value.
    var value;

To declare a variable, you can use the var, let, or const keywords followed by the variable name. var and let allow reassignment, while const is used for constants that cannot be reassigned. For example:

var age = 25;
let name = 'John';
const PI = 3.14;

It's worth noting that with the introduction of ES6 (ECMAScript 2015), let and const were introduced as block-scoped alternatives to var. Block scoping means that variables declared with let and const are only accessible within the block (enclosed within curly braces) where they are defined.
